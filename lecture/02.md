## Biến, Kiểu dữ liệu và Các toán tử

## 1. Biến (Variables) và Phép gán
Trong R, biến được dùng để lưu trữ các giá trị dữ liệu. Điểm đặc biệt của R là sử dụng toán tử mũi tên để gán giá trị.

### 1.1. Toán tử gán
* **`<-`**: Là toán tử gán chuẩn trong R (Ví dụ: `x <- 10`).
* **`=`**: Vẫn có thể sử dụng nhưng không được khuyến khích bằng `<-` trong các script chuyên nghiệp.

### 1.2. Quy tắc đặt tên biến
* Tên biến có thể bao gồm chữ cái, chữ số, dấu chấm `.` và dấu gạch dưới `_`.
* **Bắt đầu:** Phải bắt đầu bằng chữ cái hoặc dấu chấm (nếu bắt đầu bằng dấu chấm thì theo sau không được là con số).
* **Phân biệt hoa thường:** `ketQua` và `ketqua` là hai biến khác nhau.
* **Từ khóa cấm:** Không đặt tên trùng với các từ khóa hệ thống như `if`, `else`, `TRUE`, `FALSE`, ...

## 2. Các kiểu dữ liệu cơ bản (Data Types)
R phân loại dữ liệu thành các kiểu nguyên tử (atomic) sau:

| Kiểu dữ liệu | Tên gọi | Mô tả | Ví dụ |
| :--- | :--- | :--- | :--- |
| **Numeric** | Số thực | Các số có phần thập phân hoặc số nguyên | `10.5`, `3.14`, `2` |
| **Integer** | Số nguyên | Để tối ưu bộ nhớ, thêm chữ `L` sau số | `5L`, `100L` |
| **Character** | Chuỗi | Ký tự nằm trong dấu nháy đơn hoặc kép | `"Data Science"`, 'R' |
| **Logical** | Logic | Giá trị đúng hoặc sai | `TRUE` (T), `FALSE` (F) |
| **Complex** | Số phức | Số có phần ảo | `1 + 4i` |

> **Mẹo:** Sử dụng hàm `class(tên_biến)` để kiểm tra kiểu dữ liệu của biến đó.

## 3. Các toán tử (Operators)

### 3.1. Toán tử số học
Dùng để thực hiện các tính toán toán học cơ bản:
* `+` : Cộng
* `-` : Trừ
* `*` : Nhân
* `/` : Chia
* `^` hoặc `**` : Lũy thừa (Ví dụ: `2^3` kết quả là `8`)
* `%%` : Chia lấy phần dư (Modulus)
* `%/%` : Chia lấy phần nguyên (Integer division)

### 3.2. Toán tử so sánh
Kết quả trả về luôn là kiểu **Logical** (`TRUE` hoặc `FALSE`):
* `==` : So sánh bằng
* `!=` : So sánh khác
* `>`  : Lớn hơn
* `<`  : Nhỏ hơn
* `>=` : Lớn hơn hoặc bằng
* `<=` : Nhỏ hơn hoặc bằng

### 3.3. Toán tử logic
Dùng để kết hợp nhiều điều kiện so sánh:
* `&`  : Phép VÀ (AND) - Trả về TRUE khi tất cả điều kiện đúng.
* `|`  : Phép HOẶC (OR) - Trả về TRUE khi có ít nhất một điều kiện đúng.
* `!`  : Phép PHỦ ĐỊNH (NOT) - Đảo ngược giá trị logic.

## 4. Thực hành tổng hợp
Bạn hãy copy đoạn mã sau vào RStudio để chạy thử:

```R
# Bước 1: Khai báo các thông số
chieu_dai <- 20
chieu_rong <- 10
don_vi <- "met"

# Bước 2: Tính toán chu vi và diện tích
chu_vi <- (chieu_dai + chieu_rong) * 2
dien_tich <- chieu_dai * chieu_rong

# Bước 3: Sử dụng toán tử so sánh và logic
# Kiểm tra xem diện tích có lớn hơn 150 VÀ chiều dài có lớn hơn chiều rộng không
check_dk <- (dien_tich > 150) & (chieu_dai > chieu_rong)

# Bước 4: In kết quả
print(paste("Chu vi la:", chu_vi, don_vi))
print(paste("Dien tich la:", dien_tich, don_vi, "vuong"))
print(paste("Dieu kien thoa man:", check_dk))

# Bước 5: Kiểm tra kiểu dữ liệu
class(don_vi)   # Tra về "character"
class(check_dk) # Tra về "logical"
